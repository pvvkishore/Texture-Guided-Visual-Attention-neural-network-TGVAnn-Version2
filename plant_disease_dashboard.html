TGVAnn Architecture Overview

Dual-Stream Architecture with Cross-Modal Attention

üì∏ RGB Stream (Query Pathway)

Full ResNet-18 Backbone: Conv1 ‚Üí ResBlock-1 ‚Üí ResBlock-2 ‚Üí ResBlock-3 ‚Üí ResBlock-4

Purpose: Extracts appearance-based visual features (color, shape, global context)

üé® Texture Stream (Key-Value Pathway)

Partial ResNet-18: Conv1 ‚Üí ResBlock-1 ‚Üí ResBlock-2 (stops here)

Input: LDP texture features (grayscale directional patterns)

Purpose: Captures fine-grained disease-specific textural information

‚ö° TGVA Fusion Module @ ResBlock-2

Multi-Feature Projection:

  • Query (Q): RGB features from ResBlock-2
  • Key (K) & Value (V): Texture features from ResBlock-2
  • Attention: Attention = SoftMax(Q ¬∑ KT / ‚àöd_k) ¬∑ V
  • Residual Integration: TGVA = RGB_features + Attention_output

üéØ Classification Head

Dense Classifier: Global Average Pooling ‚Üí FC ‚Üí Dropout ‚Üí FC (num_classes)

Output: Disease class predictions with confidence scores

üí° Key Innovation: Multi-Feature Projection (MFP)

Unlike standard ViTs that use single-domain features for Q, K, V, TGVAnn projects heterogeneous features from two complementary domains:

  • ‚úì RGB (appearance) ‚Üí Query pathway
  • ‚úì Texture (LDP patterns) ‚Üí Key-Value pathway
  • ‚úì Preserves modality-specific characteristics while enabling cross-domain learning
  • ‚úì Superior to early fusion (loses modality info) and late fusion (no inter-modal dependencies)
